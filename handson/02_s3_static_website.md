- バケットポリシーで公開設定する

## Step 1：HTMLファイルを準備

### ローカルPCで以下のファイルを作成

#### `index.html`
```html
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>My First AWS Site</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>Hello, AWS!</h1>
    <p>これは S3 でホスティングした静的サイトです。</p>
    <p>CCP合格に向けて勉強中！</p>
</body>
</html>
```

#### `style.css`
```css
body {
    font-family: 'Helvetica', sans-serif;
    background: #232f3e;
    color: #ff9900;
    text-align: center;
    padding: 50px;
}
h1 {
    font-size: 48px;
}
```

---

## Step 2：S3バケットを作成

### 2-1. S3コンソールへ
1. AWSマネジメントコンソールで「**S3**」を検索
2. 「**バケットを作成**」をクリック

### 2-2. バケット名を決める
- 名前：`my-ccp-site-yourname-20260426`（**全世界で一意**）
- 例：`my-ccp-site-tora-20260426`

⚠️ バケット名は**全世界で重複不可**。日付や名前を含めて一意にする。

### 2-3. リージョンを選択
- 「**アジアパシフィック（東京）ap-northeast-1**」を選択

### 2-4. パブリックアクセスの設定
**「パブリックアクセスをすべてブロック」のチェックを外す**

⚠️ 通常はブロック推奨だが、**今回はWebサイトとして公開**するので外す

- 警告が表示されたら ✅ チェックを入れて承認

### 2-5. その他はデフォルトで OK
- バケットのバージョニング：無効
- 暗号化：デフォルト
- 「**バケットを作成**」をクリック

---

## Step 3：ファイルをアップロード

### 3-1. バケットを開く
作成したバケット名をクリック

### 3-2. ファイルをアップロード
1. 「**アップロード**」ボタンをクリック
2. 「**ファイルを追加**」で `index.html` と `style.css` を選択
3. 「**アップロード**」をクリック

---

## Step 4：静的Webホスティングを有効化

### 4-1. プロパティタブへ
バケット → 「**プロパティ**」タブ

### 4-2. 静的ウェブサイトホスティング
最下部の「**静的ウェブサイトホスティング**」 → 「**編集**」

設定：
- 静的ウェブサイトホスティング：**有効にする**
- ホスティングタイプ：**静的ウェブサイトをホストする**
- インデックスドキュメント：`index.html`
- エラードキュメント：`index.html`（任意）

「変更の保存」をクリック

### 4-3. エンドポイントURLをメモ
プロパティ → 静的ウェブサイトホスティングセクションに表示されるURL：
```
http://my-ccp-site-yourname-20260426.s3-website-ap-northeast-1.amazonaws.com
```

---

## Step 5：バケットポリシーで公開設定

⚠️ ここでまだアクセスすると **403 Forbidden** エラーが出る。
バケットポリシーで「読み取り許可」を明示する必要がある。

### 5-1. アクセス許可タブへ
バケット → 「**アクセス許可**」タブ

### 5-2. バケットポリシーを編集
「**バケットポリシー**」 → 「**編集**」

以下を入力（**バケット名は自分のものに置き換え**）：

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::my-ccp-site-yourname-20260426/*"
        }
    ]
}
```

### 5-3. ポリシーの解説

| 項目 | 意味 |
|------|------|
| **Effect: Allow** | 許可する |
| **Principal: "\*"** | **誰でも**（パブリック） |
| **Action: s3:GetObject** | オブジェクトの**読み取り**のみ |
| **Resource: ...** | このバケット内の**全オブジェクト** |

「変更の保存」をクリック

---

## Step 6：サイトにアクセス

### 6-1. ブラウザで確認
Step 4-3でメモしたURLにアクセス：
```
http://my-ccp-site-yourname-20260426.s3-website-ap-northeast-1.amazonaws.com
```

### ✅ 成功
オレンジ文字で「Hello, AWS!」が表示されればOK！

## 🎓 学習ポイントまとめ

### S3の特徴

| 特徴 | 内容 |
|------|------|
| **オブジェクトストレージ** | ファイル単位で管理 |
| **無制限の容量** | 必要なだけ保存可能 |
| **99.999999999%（イレブンナイン）の耐久性** | データ消失リスク極小 |
| **複数のストレージクラス** | Standard、Glacier、Intelligent-Tieringなど |

### バケットポリシー vs IAMポリシー

| 種類 | 設定対象 | 例 |
|------|--------|------|
| **バケットポリシー** | バケットに対して | 「このバケットは誰でも読める」 |
| **IAMポリシー** | ユーザー・ロールに対して | 「このユーザーはS3を操作できる」 |

### アクセス制御の優先順位

```
1. パブリックアクセスブロック（最優先）
   ↓
2. バケットポリシー
   ↓
3. IAMポリシー
   ↓
4. ACL（アクセスコントロールリスト）
```

## 💡 応用ポイント

### CloudFrontと組み合わせるとさらに高度

```
ユーザー → CloudFront（CDN） → S3
              ↓
     ・HTTPS対応
     ・世界中で高速配信
     ・キャッシュで高速化
```

CCP試験では「**世界中で高速配信したい場合は CloudFront**」と覚える。

## 🧹 後片付け

### 課金を防ぐため、必ず削除

#### ファイルの削除
1. バケットを開く
2. すべてのファイルを選択
3. 「削除」 → 「永久に削除」

#### バケットの削除
1. S3コンソール → バケット一覧
2. 該当バケットを選択
3. 「削除」 → バケット名を入力 → 「削除」

⚠️ **バケットは中身が空でないと削除できない**

| 問題例 | 答え |
|--------|------|
| 静的Webサイトのホスティングに最適なサービスは？ | **S3** |
