# ハンズオン③：EC2でWebサーバーを構築する

## 学習目標
- EC2インスタンスを起動する
- セキュリティグループを設定する
- SSHで接続する
- Webサーバー（Apache）をインストール
- ブラウザからアクセス確認

## 学べるCCP概念
- ✅ EC2（仮想サーバー）
- ✅ AMI（Amazon Machine Image）
- ✅ インスタンスタイプ
- ✅ キーペア（SSH認証）
- ✅ セキュリティグループ（仮想ファイアウォール）
- ✅ パブリックIP / プライベートIP

---

## 所要時間
約1時間

## コスト
- **t2.micro / t3.micro = 無料枠**（750時間/月）
- 必ず**使い終わったら停止 or 削除**

## Step 1：キーペアを作成

### 何のため？
- EC2にSSH接続するための**鍵**
- これがないと **SSHログインできない**

### 1-1. EC2コンソールへ
1. AWSコンソールで「**EC2**」を検索
2. 左メニュー「**ネットワーク&セキュリティ**」 → 「**キーペア**」

### 1-2. キーペアを作成
1. 「**キーペアを作成**」をクリック
2. 名前：`my-ccp-key`
3. キーペアタイプ：**RSA**
4. プライベートキーファイル形式：
   - **Mac/Linux**：`.pem`
   - **Windows（PuTTY使用）**：`.ppk`
5. 「キーペアを作成」をクリック

### 1-3. ダウンロードした鍵を安全な場所に保存
```bash
# Macの場合
mv ~/Downloads/my-ccp-key.pem ~/.ssh/
chmod 400 ~/.ssh/my-ccp-key.pem
```

⚠️ この鍵は **再ダウンロードできない**。失くしたらインスタンスにアクセス不可

---

## Step 2：EC2インスタンスを起動

### 2-1. インスタンスを起動
1. EC2コンソール → 「**インスタンスを起動**」

### 2-2. 名前
- 名前：`my-ccp-webserver`

### 2-3. AMI（OSイメージ）を選択
- **Amazon Linux 2023**（無料枠対象 ✓）

### 🎯 学習ポイント：AMIとは？
**A**mazon **M**achine **I**mage = OSとソフトウェアのテンプレート

```
AMIから起動 → OSがプリインストールされた状態でEC2が立ち上がる
```

### 2-4. インスタンスタイプ
- **t2.micro** または **t3.micro**（無料枠 ✓）
- 1 vCPU、1 GiB メモリ

### 🎯 学習ポイント：インスタンスタイプ命名規則

```
t2.micro
│ │  │
│ │  └─ サイズ（micro, small, medium, large...）
│ └──── 世代（数字、新しいほど高性能・低価格）
└────── ファミリー（用途別）
```

| ファミリー | 用途 |
|----------|------|
| **t** | バースト可能（小規模・断続的） |
| **m** | 汎用 |
| **c** | コンピューティング最適化 |
| **r** | メモリ最適化 |
| **g** | GPU |

### 2-5. キーペア
- 先ほど作成した「`my-ccp-key`」を選択

### 2-6. ネットワーク設定
**「編集」** をクリック

#### セキュリティグループ
- 「**セキュリティグループを作成する**」
- 名前：`my-ccp-webserver-sg`
- 説明：`Web Server SG`

#### インバウンドルール
| タイプ | プロトコル | ポート | ソース | 説明 |
|--------|----------|-------|------|------|
| SSH | TCP | 22 | **マイIP** | 自分だけ SSH接続可能 |
| HTTP | TCP | 80 | **任意の場所**（0.0.0.0/0） | 誰でも HTTP アクセス可能 |

⚠️ SSHを「Anywhere」にしないこと（**世界中から攻撃される**）

### 2-7. ストレージ
- 8 GB（**無料枠は30GBまで**）

### 2-8. 起動
- 「インスタンスを起動」をクリック
- 数分待つと「**実行中**」になる

## Step 3：SSHで接続

### 3-1. パブリックIPを確認
インスタンス一覧 → インスタンスを選択 → 「パブリック IPv4 アドレス」をメモ

### 3-2. ターミナルで接続

#### Mac/Linux
```bash
ssh -i ~/.ssh/my-ccp-key.pem ec2-user@[パブリックIP]
```

例：
```bash
ssh -i ~/.ssh/my-ccp-key.pem ec2-user@52.197.123.45
```

#### 初回接続時
```
Are you sure you want to continue connecting (yes/no)? yes
```

## Step 4：Apacheをインストール

### 4-1. パッケージ更新
```bash
sudo dnf update -y
```

### 4-2. Apacheインストール
```bash
sudo dnf install -y httpd
```

### 4-3. Apache起動
```bash
sudo systemctl start httpd
sudo systemctl enable httpd  # 再起動時も自動起動
```

### 4-4. 状態確認
```bash
sudo systemctl status httpd
```

`active (running)` と表示されればOK

## 🎓 学習ポイントまとめ

### EC2 = 仮想サーバー

```
1台の物理サーバー
       ↓
ハイパーバイザーで分割
       ↓
複数の仮想サーバー（EC2インスタンス）
```

### セキュリティグループとは？

🛡️ **EC2の前に立つ仮想ファイアウォール**

| 設定 | 内容 |
|------|------|
| **インバウンド** | 入ってくる通信のルール |
| **アウトバウンド** | 出て行く通信のルール（デフォルト全許可） |
| **ステートフル** | 戻りの通信は自動で許可 |

### キーペアの仕組み

```
ペアの鍵：
  公開鍵（パブリックキー） → AWSが保管（EC2上）
  秘密鍵（プライベートキー）→ 自分が保管（ローカル）

接続時：
  秘密鍵で署名 → 公開鍵で検証 → 認証成功
```

➡️ **秘密鍵がないとログインできない**

### パブリックIP vs プライベートIP

| 種類 | 用途 |
|------|------|
| **パブリックIP** | インターネット側からアクセスする時のIP |
| **プライベートIP** | VPC内部での通信用 |

⚠️ EC2を**停止 → 起動**するとパブリックIPは変わる
（固定したい場合は **Elastic IP**）

## 🧹 後片付け（重要）

### EC2インスタンスを削除

⚠️ 起動したまま放置すると、無料枠を超えて課金される

#### 方法1：停止（後で再開できる）
- インスタンス選択 → 「インスタンスの状態」 → 「停止」
- ⚠️ 停止中もEBSストレージ料金は発生

#### 方法2：終了（完全削除・推奨）
- インスタンス選択 → 「インスタンスの状態」 → 「インスタンスを終了」
- インスタンスとEBSが**完全に削除される**

### セキュリティグループの削除（任意）
- 「セキュリティグループ」 → `my-ccp-webserver-sg` を選択 → 削除

### キーペアの削除（任意）
- 削除してもよいが、再利用したいなら残してもOK

## 💡 トラブルシューティング

### SSH接続できない
- セキュリティグループでSSH(22)が**マイIP**で許可されているか
- 鍵ファイルの権限：`chmod 400 my-ccp-key.pem`
- 接続先ユーザーは `ec2-user`（Amazon Linuxの場合）

### ブラウザでアクセスできない
- セキュリティグループでHTTP(80)が「**0.0.0.0/0**」で許可されているか
- Apacheが起動しているか：`sudo systemctl status httpd`
- HTTPSではなくHTTPでアクセスしているか
