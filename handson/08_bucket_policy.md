## Step 0：バケットポリシーって何？（座学）

### 一言で

📜 **「このS3バケットに、誰が、何をしていいか」を書いた紙**

---

### バケットポリシーの構造

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}
```

### 各項目の意味

| 項目 | 読み方 | 意味 | 例 |
|------|--------|------|-----|
| **Version** | バージョン | ポリシーの書式バージョン | `2012-10-17` 固定 |
| **Statement** | ステートメント | ルールの本体（複数可） | 配列 |
| **Effect** | エフェクト | **許可するか拒否するか** | `Allow` / `Deny` |
| **Principal** | プリンシパル | **誰に対して** | `*`（全員）/ ユーザー |
| **Action** | アクション | **何をする操作** | `s3:GetObject`（読取） |
| **Resource** | リソース | **どのバケット・ファイル** | バケットのARN |

---

### 身近な例

🏠 **家のルールに例えると**

```json
{
  "Statement": [{
    "Effect": "Allow",          ← 許可する
    "Principal": "家族",         ← 誰が？
    "Action": "冷蔵庫を開ける",   ← 何をする？
    "Resource": "キッチン"       ← どこで？
  }]
}
```

➡️ 「**キッチンの冷蔵庫を、家族が開けることを許可する**」

## Step 1：S3バケットを作成（5分）

### 1-1. S3コンソールへアクセス
1. AWSコンソールで「**S3**」を検索
2. 「**バケットを作成**」をクリック

### 1-2. 設定
- バケット名：`bucket-policy-test-yourname-20260428`
  - ⚠️ 全世界で一意。自分の名前や日付を入れる
- リージョン：**東京（ap-northeast-1）**
- 「**パブリックアクセスをすべてブロック**」のチェックを**外す**
  - 確認のチェックボックスにもチェック

### 1-3. その他はデフォルト
- 「バケットを作成」をクリック

✅ バケット作成完了

## Step 2：ファイルをアップロード → アクセスできない！を体感（10分）

### 2-1. テスト用HTMLファイルを準備

ローカルPCで `test.html` を作成：

```html
<!DOCTYPE html>
<html>
<body style="background:#232f3e; color:#ff9900; text-align:center; padding:50px;">
  <h1>バケットポリシーのテスト</h1>
  <p>このファイルが見えればアクセス成功！</p>
</body>
</html>
```

### 2-2. アップロード
1. 作成したバケットを開く
2. 「アップロード」 → ファイル追加
3. `test.html` をアップロード

### 2-3. ファイルのURLを確認
1. アップロードした `test.html` をクリック
2. 「**オブジェクトURL**」をコピー
   - 例：`https://bucket-policy-test-yourname-20260428.s3.ap-northeast-1.amazonaws.com/test.html`

### 2-4. ブラウザでアクセス
ブラウザにURLを貼り付ける

### ❌ 結果：エラーが出る！

```xml
<Error>
  <Code>AccessDenied</Code>
  <Message>Access Denied</Message>
</Error>
```

### 🤔 なぜ？

- バケットはあるが、**「誰がアクセスしていいか」のルールが未設定**
- S3はデフォルトで**「拒否」**
- アクセスを許可するには **バケットポリシーが必要**

## Step 3：バケットポリシーで「全員に公開」（15分）

### 3-1. バケットポリシーを開く
1. バケット → 「**アクセス許可**」タブ
2. 「**バケットポリシー**」セクション → 「**編集**」

### 3-2. 以下のJSONを貼り付け（バケット名を自分のものに）

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowPublicRead",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::bucket-policy-test-yourname-20260428/*"
    }
  ]
}
```

### 3-3. JSON 1行ずつ解説

| 行 | 内容 | 解説 |
|---|------|------|
| `"Sid": "AllowPublicRead"` | ルールの名前 | 自分用のメモ（任意） |
| `"Effect": "Allow"` | **許可する** | DenyにすればI拒否 |
| `"Principal": "*"` | **誰でも** | `*` = 全世界 |
| `"Action": "s3:GetObject"` | **読み取り操作** | `s3:` はS3のアクション |
| `"Resource": "arn:...:bucket-name/*"` | **このバケット内の全ファイル** | `/*` = 全オブジェクト |

### 3-4. 保存
「変更の保存」をクリック

### 3-5. もう一度ブラウザでアクセス

✅ **HTMLが表示される！**

オレンジ文字で「バケットポリシーのテスト」が見える

### 🎉 体感ポイント

```
バケットポリシーがない → ❌ 拒否（デフォルト）
バケットポリシーで許可 → ✅ アクセス可能
```

## Step 4：「特定のIPだけ許可」に変更（15分）

### 4-1. 自分のグローバルIPを調べる

ブラウザで以下にアクセス：
```
https://api.ipify.org
```

数字が表示される（例：`203.0.113.45`）→ これが**あなたのグローバルIP**

### 4-2. バケットポリシーを書き換え

「アクセス許可」 → 「バケットポリシー」 → 「編集」

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowFromMyIP",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::bucket-policy-test-yourname-20260428/*",
      "Condition": {
        "IpAddress": {
          "aws:SourceIp": "203.0.113.45/32"
        }
      }
    }
  ]
}
```

⚠️ `203.0.113.45` は自分のIPに置き換える

### 4-3. 新しい要素：Condition（条件）

| 項目 | 内容 |
|------|------|
| **Condition** | 「○○の場合のみ」という条件 |
| **IpAddress** | IPアドレスでの判定 |
| **aws:SourceIp** | アクセス元のIP |
| `/32` | このIP1個だけ（範囲指定可） |

### 4-4. 動作確認

#### 自分のPC（許可されたIP）
```
✅ ブラウザでアクセス → HTML表示される
```

#### スマホ（4G/5G通信）からアクセス
```
❌ Access Denied
```

→ **同じURLなのに、IPが違うとアクセスできない**

---

### 4-5. 体感ポイント

```
誰でも読める → 特定のIPだけ読める
```

これで「**バケットポリシーは細かく制御できる**」を体感できる。

## Step 5：「特定のIAMユーザーのみ許可」に変更（10分）

### 5-1. IAMユーザーを作成（まだなければ）

1. IAMコンソール → 「ユーザー」 → 「ユーザーを作成」
2. ユーザー名：`s3-test-user`
3. アクセス権限：**何もアタッチしない**（バケットポリシーで制御）
4. 作成

### 5-2. ユーザーのARNをコピー

ユーザー詳細画面の上部に表示：
```
arn:aws:iam::123456789012:user/s3-test-user
```

### 5-3. バケットポリシーを書き換え

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowSpecificUser",
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::123456789012:user/s3-test-user"
      },
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::bucket-policy-test-yourname-20260428/*"
    }
  ]
}
```

⚠️ ARNは自分のユーザーのものに置き換える

### 5-4. 違いを確認

| 項目 | 全員 | 特定ユーザー |
|------|------|-----------|
| `Principal` | `"*"` | `{"AWS": "arn:..."}` |

### 5-5. 動作確認

#### ブラウザ（認証なし）
```
❌ Access Denied
```

#### `s3-test-user`でCLI実行
```bash
aws s3 cp s3://bucket-policy-test-yourname-20260428/test.html ./
```
```
✅ ダウンロード成功
```

## Step 6：拒否（Deny）を体験（応用）

### 6-1. 「全員許可、ただし特定IPは拒否」

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowAll",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::bucket-policy-test-yourname-20260428/*"
    },
    {
      "Sid": "DenyBadIP",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::bucket-policy-test-yourname-20260428/*",
      "Condition": {
        "IpAddress": {
          "aws:SourceIp": "1.2.3.4/32"
        }
      }
    }
  ]
}
```

### 重要ルール：**Deny は Allow より強い**

```
全員Allow + 特定IP Deny = 特定IP以外がアクセス可
```

## 🎓 学習ポイントまとめ

### バケットポリシーの構造

```
{
  "Statement": [
    {
      Effect    : Allow / Deny
      Principal : 誰が
      Action    : 何をする
      Resource  : どこで
      Condition : いつ・どんな時
    }
  ]
}
```

### 5W1Hで覚える

| 5W1H | ポリシーの項目 |
|------|------------|
| **誰が（Who）** | Principal |
| **何を（What）** | Action |
| **どこで（Where）** | Resource |
| **いつ（When）** | Condition |
| **どうする（How）** | Effect（Allow/Deny） |

---

### 評価ロジック（重要）

```
1. 明示的な Deny がある？ → 拒否（最優先）
2. 明示的な Allow がある？ → 許可
3. どちらもない？ → 拒否（デフォルト）
```

➡️ **「Deny > Allow > デフォルト拒否」**

| 問題例 | 答え |
|--------|------|
| S3を**全世界に公開**したい | バケットポリシーで `Principal: "*"` |
| 特定の**IAMユーザーだけ**許可 | バケットポリシーで Principal に ARN指定 |
| **特定IPだけ**許可 | Condition で `aws:SourceIp` |
| Deny と Allow が両方ある場合は？ | **Deny が優先** |
| デフォルトは？ | **拒否** |

## 💡 イメージが定着するコツ

実際に**手を動かす**のが一番大事。

### 提案する体験順
```
1. アクセスできない（デフォルト拒否）を体感
   ↓
2. Allow ですべて許可 → 体感
   ↓
3. Condition で範囲を絞る → 体感
   ↓
4. Deny を組み合わせる → 体感
```

各段階で「**ブラウザで実際にアクセスしてみる**」と、ポリシーの効果が肌でわかる。
